## Goal
Resolve the reported nvim-treesitter health errors by making parser/query paths discoverable via runtimepath and documenting the required external CLI dependency.

## Confirmed Choice
- Keep `auto_install = true` for nvim-treesitter.
- Satisfy the CLI prerequisite by installing `tree-sitter` (`>= 0.26.1`).

## Recommended Approach
1. Update lazy bootstrap runtimepath handling.
   - File: `lua/config/lazy.lua`
   - Add logic after `vim.opt.rtp:prepend(lazypath)` to append `vim.fn.stdpath("data") .. "/site"` to `runtimepath` if not already present.
   - Why: health output shows parser/query install directory is under `~/.local/share/nvim/site` but currently not in `runtimepath`.

2. Make treesitter parser install directory explicit and aligned.
   - File: `lua/plugins/nvim-treesitter.lua`
   - Define `local parser_install_dir = vim.fn.stdpath("data") .. "/site"` in config scope.
   - Pass `parser_install_dir = parser_install_dir` into `require("nvim-treesitter.configs").setup({...})`.
   - Keep `auto_install = true` as-is (current behavior).
   - Add a short note that parser install dir must be on runtimepath.

3. Preserve current plugin behavior; avoid unrelated refactors.
   - No changes to plugin manager choice or plugin loading strategy.
   - No forced parser reinstall in code changes.

4. Document unavoidable external prerequisite.
   - `tree-sitter` CLI must be installed on host (`>= 0.26.1`) to satisfy health requirement and parser builds.
   - For macOS arm64, prefer Homebrew arm64 binary and ensure Neovim PATH can find it.

## Files To Modify
- `lua/config/lazy.lua`
- `lua/plugins/nvim-treesitter.lua`

## Verification Plan
1. Startup validation:
   - `nvim --headless -u init.lua -c 'quit'`

2. Runtimepath check:
   - In Neovim: `:echo &runtimepath`
   - Confirm it includes `vim.fn.stdpath("data") .. "/site"`.

3. Treesitter health check:
   - In Neovim: `:checkhealth nvim-treesitter`
   - Confirm the previous error about install dir not in runtimepath is gone.
   - If CLI was missing before, confirm `tree-sitter-cli` requirement is satisfied after installation.

4. Parser install smoke test:
   - In Neovim: `:TSInstallSync lua`
   - Confirm parser installs/loads without runtimepath-related warnings.

## Notes / Edge Cases

- If parser binaries were built under mismatched architecture (Rosetta vs arm64), reinstall parser(s) after fixing toolchain/path.
