# Fix lazy.nvim Tree-sitter Spec Load Error

## Goal
Resolve startup failure:
`module 'nvim-treesitter.configs' not found`

User-selected outcome: **Enable Tree-sitter**.

## Recommended approach
1. Update `lua/plugins/nvim-treesitter.lua` so Tree-sitter is enabled.
2. Move Tree-sitter setup call out of top-level evaluation and into Lazy's plugin load lifecycle.
3. Keep existing Tree-sitter options unchanged; only relocate them into `config = function()` (or `opts` + `config(_, opts)`).

## Why this works
- `lua/config/lazy.lua` imports `plugins`, so each spec file is evaluated during startup.
- A top-level `require("nvim-treesitter.configs")` runs too early and can fail before plugin runtime paths are available.
- Deferring `require(...).setup(...)` to `config` ensures it runs when the plugin is loaded.

## Files to modify
- `lua/plugins/nvim-treesitter.lua`
  - Change `enabled = false` to `enabled = true`.
  - Remove top-level `require("nvim-treesitter.configs").setup(...)` execution.
  - Add/keep setup under `config = function()`:
    - `require("nvim-treesitter.configs").setup({ ...existing options... })`

## Verification
1. Headless startup check:
   - `nvim --headless -u init.lua -c 'quit'`
2. Ensure plugin install/update if needed:
   - `nvim --headless -u init.lua -c 'Lazy! sync' -c 'quitall'`
3. Confirm no startup error references `nvim-treesitter.configs`.
4. Optional style/lint checks from repo guidance:
   - `stylua lua/plugins/nvim-treesitter.lua`
   - `luacheck . --std lua52`

## Notes
- Keep changes minimal and limited to Tree-sitter spec behavior.
- Do not alter unrelated plugin files unless verification reveals a direct dependency issue.
