# Plan: Fix external-tool checks and Treesitter coverage

Goal
----
Bring the Neovim config into a reproducible state by:
- documenting and checking required external binaries (mmdc, ImageMagick/magick/convert, pdflatex, tree-sitter CLI, lazygit)
- ensuring Treesitter installs/parsers cover the languages the user expects and documenting missing parsers
- adding lightweight verification steps to validate the config headlessly

Scope
-----
This plan touches only configuration and docs. Changes will be made to plugin config files and docs under `lua/plugins/` and `AGENTS.md` (and possibly `README.md`). Critical files identified during exploration:

- `init.lua`
- `lua/plugins/nvim-treesitter.lua` (# ensure_installed, parser_install_dir)
- `lua/plugins/snacks.lua` (mappings referencing lazygit)
- `AGENTS.md`, `README.md` (docs for external tools)

Findings (from repo scan)
-------------------------
- Treesitter: `lua/plugins/nvim-treesitter.lua` already sets `ensure_installed` to a small list (c, lua, vim, cpp, python) and has some ignored parsers.
- External tools: lazygit is referenced by `lua/plugins/snacks.lua`. No occurrences of `mmdc`, `magick`/`convert`, or `pdflatex` were found in code, but the runtime diagnostics mention them (mmdc missing, magick missing). That suggests docs or runtime checks outside the repo surfaced the diagnostics.

Recommended changes (implementation steps)
-----------------------------------------
1) Add a small `lua/.opencode/scripts/check-binaries.lua` (or inline code in an existing plugin loader) that:
   - Checks for required binaries: `mmdc`, `magick` (or `convert`), `pdflatex`, `tree-sitter`, `lazygit`, `gs`, `tmux` (if used).
   - Emits a non-fatal `vim.notify()` with advice and links when a binary is missing.
   - Keep the check opt-in (call it from a single `require('opencode').verify()` in `init.lua` or leave commented-examples in docs).

2) Update documentation (`AGENTS.md` and `README.md`) to list these external requirements explicitly and provide quick install commands for common OSes (macOS brew, Debian apt).

3) Update `lua/plugins/nvim-treesitter.lua`:
   - Expand `ensure_installed` to include commonly-missing languages (css, html, javascript, latex, norg, scss, svelte, tsx, typst, vue) or document why they are omitted.
   - Ensure `parser_install_dir` is set to `vim.fn.stdpath('data') .. '/site'` and document the need for the `tree-sitter` CLI when building parsers.

4) Add a small headless verification task to the docs and optionally as a `Makefile` or `scripts/validate.sh` that runs:
   - `stylua .` (format check)
   - `luacheck . --std lua52` (if available)
   - `nvim --headless -u init.lua -c 'quit'` to surface runtime errors
   - `command -v mmdc || echo "mmdc missing"` etc. (or the Lua check script)

5) (Optional) If Mermaid (`mmdc`) and ImageMagick are critical to any doc rendering pipeline, gate their use so missing programs fall back to static PNGs or emit clear messages.

Critical files to modify
------------------------
- `lua/plugins/nvim-treesitter.lua`
- `lua/plugins/snacks.lua` (if adding/adjusting checks or mapping descriptions)
- `AGENTS.md` and `README.md` (docs update)
- optionally add `lua/opencode/verify.lua` or `lua/.opencode/scripts/check-binaries.lua`

Verification (how to test end-to-end)
-----------------------------------
1) Run formatting and linting: `stylua .` and `luacheck . --std lua52` (install if needed).
2) Validate Neovim startup headless: `nvim --headless -u init.lua -c 'quit'` — confirm no runtime errors.
3) Run the binary checker (Lua or shell) and confirm it reports missing tools with helpful messages.
4) From inside Neovim, trigger any mappings added/changed (e.g., lazygit mapping) to ensure they still work.

Notes & Risks

- If you prefer automated installs of missing CLIs, we can add install hints per platform but avoid automatic installs in the repo.
